# Rencana: Arsipkan dan Hapus File Admin/Dashboard (Pertahankan Landing Page)

Tujuan
- Arsipkan semua file dan route terkait admin/dashboard yang tidak diperlukan untuk landing page.
- Hapus dependensi yang hanya dipakai oleh dashboard dari package.json.
- Pastikan landing page (`src/app/landing`) tetap berfungsi penuh.

Keputusan Pengguna (sudah ditetapkan)
- Hapus dependensi dashboard-only sekarang: YA
- Arsipkan file ke `archive/dashboard/` (tidak dihapus permanen): YA

Keamanan & Prinsip Kerja
- Semua perubahan dilakukan di branch baru: `chore/remove-dashboard-files`.
- File yang diarsipkan dipindahkan dengan `git mv` ke `archive/dashboard/<original-path>` agar history tetap ada dan rollback mudah.
- Jangan hapus primitives atau util bersama yang masih digunakan landing page; verifikasi penggunaan sebelum menghapus.
- Buat snapshot commit sebelum perubahan destruktif.

Langkah Persiapan (analisis baca-saja sebelum implementasi)
1. Buat branch dan commit snapshot:
   - git switch -c chore/remove-dashboard-files
   - git add -A && git commit -m "chore: snapshot sebelum mengarsipkan dashboard"
2. Temukan referensi/dashboard-only:
   - rg "app-sidebar|nav-main|nav-user|sidebar-notification|dashboard|/dashboard" -n src
3. Periksa file besar dan penggunaan:
   - ls -la src/components | sort -k5 -n
   - rg "src/components/(app-sidebar|nav-main|nav-user|sidebar-notification|command-search)" -n || true
4. Jalankan pengecekan tipe & lint:
   - npx tsc --noEmit
   - npm run lint

Daftar Kandidat Untuk Diarsipkan (akhirnya diverifikasi dengan ripgrep sebelum diubah)
- src/components/app-sidebar.tsx
- src/components/nav-main.tsx
- src/components/nav-user.tsx
- src/components/sidebar-notification.tsx
- src/components/layouts/ (folder jika khusus dashboard)
- src/components/theme-customizer/ (folder jika khusus dashboard)
- src/components/nav-secondary.tsx (jika dashboard-only)
- src/components/pricing-plans.tsx (jika hanya untuk dashboard)
- src/components/upgrade-to-pro-button.tsx (dashboard-only)
- src/contexts/sidebar-context.tsx (arsip jika landing tidak memakainya)
- src/app/dashboard/ (route folder jika ada)
- src/app/settings/ (route folder jika hanya untuk admin)
- API routes di src/app/api/ yang hanya relevan untuk dashboard

Langkah Implementasi (aman, urut)
1. Buat branch dan snapshot commit (lihat bagian Persiapan).
2. Jalankan ripgrep untuk daftar pasti file yang akan diarsipkan.
3. Untuk tiap file/folder dashboard-only:
   a. Buat folder arsip: `mkdir -p archive/dashboard/$(dirname <path>)`.
   b. `git mv <path> archive/dashboard/<path>` (memindahkan dengan git, menjaga histori).
   c. Jalankan `npx tsc --noEmit` untuk mengecek import yang hilang.
   d. Perbaiki import yang tersisa (hapus atau arahkan ulang) di file yang masih aktif.
   e. Jalankan `npm run lint` dan `npm run build` untuk verifikasi.
4. Sesuaikan `src/app/layout.tsx` jika perlu:
   - Hapus wrapper/provider yang hanya dipakai dashboard (misal SidebarConfigProvider) dan pertahankan ThemeProvider jika landing membutuhkannya.
5. Hapus dependensi dashboard-only dari package.json:
   a. Kandidat (verifikasi penggunaan): `recharts`, `@dnd-kit/*`, `zustand`, `@tanstack/react-table`, dll.
   b. Edit package.json untuk menghapus paket tidak terpakai.
   c. Jalankan `npm install`/`npm ci` untuk memperbarui node_modules.
   d. Jalankan `npm run build` dan `npx tsc --noEmit` untuk memastikan tidak ada missing types.
6. QA akhir:
   - Jalankan `npm run dev` dan buka landing page `/landing`.
   - Verifikasi interaksi utama: navbar (desktop & mobile), smooth scroll, hero CTA, footer, theme toggle.
   - Jalankan `npx tsc --noEmit`, `npm run lint`, `npm run build`.
7. Commit & push:
   - `git add -A && git commit -m "chore: archive dashboard files; remove dashboard-only deps"`
   - Push branch dan buat PR untuk review.

Rollback
- Karena pemindahan memakai `git mv`, rollback mudah dengan revert commit atau `git mv` balik dari `archive/dashboard/`.

Deliverable Setelah Implementasi
- Branch dengan file yang dipindah ke `archive/dashboard/`.
- Patch yang menghapus dependensi dashboard-only dari package.json.
- Laporan singkat: daftar file yang dipindah dan dependensi yang dihapus.
